# HubSpot Logging: The Complete Guide

## Training Module for 1-800-Packouts SDR Onboarding

This document explains why and how to log every call in HubSpot. HubSpot is the company's memory. If a call does not have a note, it did not happen. No exceptions.

---

## Why Every Call Gets a Note

Matt uses the SDR's HubSpot notes to:
- Decide which leads to follow up on personally
- Prepare for partnership conversations with GCs and adjusters
- Track pipeline and activity levels
- Spot-check the quality of calls and conversations

If a call does not have a note, it did not happen. This is non-negotiable.

## The Note Template

Every call gets a note using this exact template:

Call Type: [Fire Lead / Cold - GC / Cold - Adjuster / Cold - PM / Follow-Up]
Outcome: [Live Conversation / Voicemail / No Answer / Wrong Number / Gatekeeper]
Contact: [Name, Title if known]
Company: [Company name if applicable]

Intel Gathered:
- Current packout vendor: [name or "none"]
- Volume: [X jobs/month or "unknown"]
- Contract status: [locked in / job-by-job / unknown]
- Decision maker: [name + title]
- Insurance carrier: [name if applicable]
- Adjuster: [name if applicable]
- Interest level: [cold / lukewarm / warm / hot]

Next Steps:
- [Specific action + date, e.g., "Follow up Wednesday 3/19" or "Escalate to Matt — warm lead"]

Fill in what was learned. If a field does not apply (for example, "Insurance carrier" on a GC call), skip it. But the structure stays the same every time.

## Contact Property Updates

After logging the note, also update the contact record:

- Lead Status: Update after every interaction. The progression is New → Attempted to Contact → Connected → Qualified, and so on.
- Phone: If they gave a better number, update it.
- Company: Associate the contact with a company record if applicable.
- Notes field on the contact: Keep this for quick summary information, not call logs.

## Common Mistakes to Avoid

Logging hours later: Log immediately after the call while it is fresh. Even quick notes are better than no notes.

Vague next steps: "Follow up later" is useless. "Follow up Thursday 3/20, ask about adjuster assignment" is useful. Always include a specific date and a specific reason.

Missing intel fields: Even "unknown" is valuable information. It tells Matt what needs to be dug into on a follow-up.

Not logging voicemails: Voicemails get logged too. "Left VM, follow up in 2 days."

Editing someone else's notes: Never do this. Only add your own notes. If you see an error in someone else's note, tell Matt.

## Key Takeaways

1. Every call gets a note — live conversation, voicemail, no answer, everything
2. Use the exact template every time for consistency
3. Log immediately after the call, not hours later
4. Next steps must be specific: date + action + reason
5. Even "unknown" is valuable intel — it tells Matt what to dig into
6. Never edit someone else's notes
7. Update contact properties (lead status, phone, company) after every interaction
